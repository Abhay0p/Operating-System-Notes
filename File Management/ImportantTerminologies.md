# IT Support Specialist: OS & CLI Glossary

## 1. Interface and Shell
* **GUI**: Graphical User Interface. The visual way to interact with a computer.
* **CLI**: Command Line Interpreter. A text interface that translates commands to the OS.
* **Shell**: The command line interpreter specifically for Linux.
* **Bash**: The language used to interact with the Linux shell.
* **.exe**: Windows file extension for an executable program.
* **Prompt**: The text at the start of your line showing which directory you are in.
* **Hot key**: A keyboard shortcut for a specific task (e.g., Ctrl+C).
* **Alias**: A "nickname" for a long or common command.

## 2. File System Hierarchy
* **Paths**: The location address of files and directories.
* **Root Directory**: The top-level parent for all other directories (C:\ in Windows, / in Linux).
* **Absolute path**: A path starting from the main/root directory.
* **Relative path**: A path starting from your current location.
* **Parent & Child**: A hierarchy where one directory (parent) houses another (child).
* **Subdirectories**: Any directory at a deeper level in the hierarchy.
* **Hidden files**: Files not visible by default to prevent accidental modification or for privacy.

## 3. Command Mechanics
* **Parameter**: A value or "argument" associated with a command.
* **Tab completion**: Using the Tab key to auto-complete names and commands.
* **Wildcard (*)**: A character used to select files based on a pattern.
* **Regular expression**: An advanced pattern-matching language for complex searches.
* **Escape characters**: Using a symbol (like `\ ` or `` ` ``) to treat the next character literally (e.g., to include a space in a name).
* **Appending**: Adding data to the end of a file without erasing what is already there.

## 4. Data Streams (I/O)
* **I/O Streams**: Systems that handle data flowing into and out of a program.
* **Standard In (stdin)**: Data flowing into a program (usually from the keyboard).
* **Standard Out (stdout)**: Data flowing out of a program (usually to the screen).
* **Standard Error (stderr)**: A specific stream for error messages, separate from stdout.

## 5. System Services & Security
* **Groups**: A collection of users categorized together for access/permissions.
* **Windows Search service**: A background service that indexes files and properties for faster searching.
